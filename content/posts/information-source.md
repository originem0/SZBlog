> **核心原则**：最大化 P(好信息|我看的信息)，而非 P(我看的好信息|所有好信息)
>
> **操作原则**：问题驱动 > 囤积驱动。没有在解决的问题，就没有有用的信息。
>
> **元原则**：信息的价值 = f(信息质量, 你的吸收能力, 信息与你问题的匹配度)
>

## 信息分级标准

| 层级 | 描述                    | 特征                                 | 使用场景                 |
| ---- | ----------------------- | ------------------------------------ | ------------------------ |
| T1   | 原创研究者/工具创造者   | 论文作者、教材编写者、开源项目创始人 | 深度学习、建立根基       |
| T2   | 领域科学家/资深专家本人 | 有重大贡献的学者、一线实践领袖       | 追踪前沿、获取第一手见解 |
| T3   | 专业评测/分析/教程作者  | 科学记者、深度测评者、高质量教程作者 | 快速入门、交叉验证       |
| T4   | 有丰富经验的一线实践者  | 资深从业者、独立开发者、深度实践者   | 实操指导、案例学习       |
| T5   | PhD背景学者/学生        | 博士生、博后、年轻学者               | 论文解读、学术讨论       |
| T6   | 非本专业的深度爱好者    | 跨界学习者、业余专家                 | 启发、发现新视角         |
| T7   | KPI驱动的内容生产者     | SEO文章、洗稿账号、软文              | ⚠️ 避免                   |

**使用原则**：
- 入门阶段可从 T3-T4 开始，建立基础后逐步向 T1-T2 迁移
- 同一主题交叉验证时，优先信任层级更高的来源
- T1-T2 的旧文章 > T6-T7 的新文章
- **领域差异**：社会/哲学领域，经典比前沿重要10倍；AI领域，前沿追踪更关键

#### 你应该知道的争论

```
"意识的困难问题"是否存在？
  ├─ 存在（Chalmers）→ 需要全新的物理学
  ├─ 不存在（Dennett）→ 意识只是认知过程的"用户幻觉"
  └─ 问题本身就问错了（Evan Thompson）→ 需要超越主客二分

"具身认知"的强弱之争：
  ├─ 弱版本：身体影响认知（几乎无争议）
  ├─ 中版本：认知依赖于身体（Clark、Thompson）
  └─ 强版本：认知就是身体行动（Varela 原教旨主义）
  注意：大部分争论在于中版本的边界在哪

"预测加工"能否统一认知科学？
  ├─ 乐观派（Clark, Friston）
  ├─ 温和派（Hohwy：能，但需要解释现象意识）
  └─ 怀疑派（有多位年轻学者质疑其可证伪性）
```

#### 关键方法论洞察

> **最好的 PKM 不是最好的笔记系统，而是最好的问题。**

#### 核心框架（来自80,000 Hours）

```
职业选择的三要素：
1. 个人适配度（Personal Fit）
   - 你在这个领域能达到前20%吗？
   - 技能、兴趣、价值观是否匹配？

2. 领域潜力（Problem Area）
   - 这个领域的天花板有多高？
   - 10年后这个领域会怎样？

3. 杠杆位置（Career Capital）
   - 这份工作能给你什么可迁移的资本？
   - 技能、人脉、声誉、资金

关键洞察：
- 不要只追"热情"，要追"能力+市场需求"的交集
- 职业资本 > 职业激情（Cal Newport）
- 最好的职业决策往往是"可逆的实验"而非"一次性押注"
```

#### 信息陷阱

- **成功学/鸡汤**：只讲故事不讲方法，没有可操作性
- **贩卖焦虑**：用"35岁危机""行业寒冬"制造恐慌
- **幸存者偏差**：只展示成功案例，不讨论失败概率
- **过度泛化**：把个人经验当成普适规律

```

    ↓
```

## Part 5: 审计与维护

### 个人信息陷阱诊断

> 基于你的方法论文件和阅读习惯，你容易踩的坑：

| 陷阱类型         | 你为什么容易踩                   | 解毒方法                           |
| ---------------- | -------------------------------- | ---------------------------------- |
| **精致囤积**     | 你有收集信息源的习惯             | 每个领域只保留 3 个活跃信息源      |
| **大师崇拜**     | 你的兴趣偏向思想史               | 每读一位大师，必读一篇对他的批评   |
| **中文降级偏见** | 方法论文件多次暗示英文 > 中文    | 改为"一手语言优先"                 |
| **理论 > 实证**  | 你偏好宏大叙事（齐泽克、王东岳） | 每读一个理论，问"这个能被证伪吗？" |
| **前沿焦虑**     | 方法论文件强调 bleeding edge     | **月度而非每日**追踪AI动态         |

### 信息陷阱通用警示

| 类型                | 特征                     | 如何识别                       |
| ------------------- | ------------------------ | ------------------------------ |
| AI生成的"工具合集"  | 缺乏实际测试             | 无具体使用体验、无截图、无案例 |
| SEO驱动的虚假专业性 | 占据搜索高位但无独特声音 | 缺乏真实案例研究、不讨论权衡   |
| 过时的"框架崇拜"    | 仍在推崇已过时的技术     | 忽视当前工业前沿趋势           |
| 无证据的"成功学"    | 只谈意志力               | 缺乏实验数据、引用             |
| 金句化/庸俗化       | 剥离论证过程的短视频剪辑 | 只有结论，没有推理过程         |
